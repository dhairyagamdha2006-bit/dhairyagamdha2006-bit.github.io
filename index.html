Most of my work sits across two areas: backend services (Spring Boot, FastAPI, PostgreSQL, Kafka) and the systems side underneath them — concurrency, latency, and what happens to a program under load.

Latency Anomaly Detection Research
Undergraduate Research
Feb 2026 — Present
Problem
Performance problems in concurrent systems often show up as subtle timing anomalies that are hard to catch by eye in raw telemetry.
What I built
Multithreaded test harnesses in Java and C++ that generate concurrent load and capture latency/throughput data, plus Python/Bash automation on Linux to run benchmarks and collect results. I then trained an LSTM model to flag anomalous latency patterns.
Result
The model reaches ~75% detection accuracy, measured on a held-out test set of labeled telemetry samples. I treat this as a working baseline I'm still improving, not a final number.
Hardest part
Reproducing anomalies reliably — I had to tune the load generators to surface race conditions and synchronization bottlenecks consistently enough to label and learn from.

Problem
Tracking holdings across multiple brokerages means re-entering data by hand, and every broker exports a differently-shaped CSV.
What I built
A full-stack portfolio dashboard with market search, watchlists, and performance charts, served over secure REST routes using the Alpaca market-data API. The core feature is a CSV import pipeline that normalizes holdings across multiple brokerage formats.

Healthcare Data Integration Service
Backend · Microservice
2026
Problem
Clinical data often arrives in batches that need validation and normalization before they're useful, and doing that work inside the request would make the API slow and fragile.
What I built
A Spring Boot microservice (Java 21) that ingests batched observation payloads, validates them, stores the raw message, and exposes patient-centered REST APIs — with API-key auth, health checks, structured logging, and OpenAPI docs.
Technical challenge
Keeping the API responsive under batched load. I put a Kafka pipeline between ingestion and normalization, so the endpoint accepts and acknowledges a payload, then hands the heavier processing to a consumer asynchronously instead of blocking the request.
Next
Adding stronger integration tests around the ingestion and consumer paths.
